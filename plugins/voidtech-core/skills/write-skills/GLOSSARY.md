本文件解释 [`write-skills`](SKILL.md) 使用的概念。中文名称用于日常讨论，英文名称仅用于对照 Claude Code 文档或上游资料。

## 调用与成本

### 可预测性（Predictability）

技能在不同会话中稳定遵循同一套过程。它约束工作方式，不要求每次输出完全相同。

### 模型可调用（Model-Invoked）

未设置 `disable-model-invocation: true` 的技能。Claude 能根据 `description` 自动选择它，其他技能也能通过 Skill 工具调用它。

### 仅用户调用（User-Invoked）

设置了 `disable-model-invocation: true` 的技能。它仍显示在用户菜单中，但 Claude 不会自动选择，其他技能也不能调用。

### Description

技能的机器可读摘要。允许模型调用时，它负责说明用途和触发场景；仅用户调用时，它主要作为菜单说明。

### 引用说明（Context Pointer）

告诉 agent 某份参考材料在哪里、何时需要读取的说明。说明必须具体，不能只放一个没有使用条件的链接。

### 上下文开销（Context Load）

允许模型调用的技能会把名称和 description 放进 Claude 的上下文。技能越多、description 越长，占用的上下文就越多。

### 用户记忆负担（Cognitive Load）

仅用户调用的技能需要用户自己记住名称和使用时机。数量过多时，应提供导航技能或减少拆分。

### 导航技能（Router Skill）

集中列出一组仅用户调用的技能，并说明各自适用场景。它帮助用户选择命令，但不能代替用户调用这些技能。

## 内容组织

### 粒度（Granularity）

一个技能承担多少职责。只有当触发条件或执行阶段明显不同时才拆分，避免把简单流程切得过碎。

### 信息层级（Information Hierarchy）

按使用频率安排内容：执行步骤优先，所有场景都需要的规则其次，只有部分场景需要的详细材料放到独立文件。

### 就近组织（Co-location）

把同一概念的定义、规则和注意事项放在一起，避免读者在文件中来回查找。

### 使用场景（Branch）

技能处理的一类独立情况。不同场景需要不同步骤或参考材料时，应明确分开说明。

### 渐进披露（Progressive Disclosure）

把只在特定场景使用的详细材料移到独立文件，并在 `SKILL.md` 中说明何时读取。这样可以保持主流程简洁。

### 步骤（Step）

agent 需要按顺序执行的动作。每一步都应说明做什么，并给出清楚的完成条件。

### 完成条件（Completion Criterion）

判断某一步是否真正完成的可验证标准。好的完成条件既明确，又能覆盖该步骤必须完成的工作。

### 后续步骤（Post-Completion Steps）

当前步骤完成后才应执行的动作。如果后续内容让 agent 过早跳过当前工作，应拆成单独技能或延后展示。

### 实际工作（Legwork）

agent 为完成当前步骤需要自行完成的搜索、阅读、修改和验证。技能应推动 agent 做这些工作，而不是把可自行解决的问题交还给用户。

### 参考材料（Reference）

定义、规则、事实或示例。所有执行路径都需要的材料可以留在 `SKILL.md`，其余内容按需拆出。

### 外部参考文件（External Reference）

不可单独调用、但可被多个技能读取的普通文件。适合存放共享契约、格式说明或详细背景。

### 关键词锚点（Leading Word）

简短且已有明确含义的关键词，用来统一触发条件和执行方式。优先使用团队熟悉的术语，不要为了新颖而自造词。

### 唯一来源（Single Source of Truth）

每项规则只保留一个权威位置。其他地方通过引用使用它，避免修改时需要同步多份副本。

### 相关性（Relevance）

一段内容是否仍然帮助技能完成当前任务。与任务无关或已经过时的内容应删除。

## 常见问题

### 过早结束（Premature Completion）

当前步骤尚未达到完成条件，agent 就开始执行后续步骤。优先把完成条件写清楚，必要时再拆分流程。

### 重复（Duplication）

同一含义出现在多个位置，增加维护成本，也会让它显得比实际更重要。

### 陈旧内容（Sediment）

技能长期只增加内容、不清理旧规则，导致过时信息逐渐掩盖有效流程。

### 内容过长（Sprawl）

即使每句话都正确，文件过长仍会降低可读性。把只在特定场景使用的材料移到独立文件。

### 无效指令（No-Op）

只是重复模型默认行为、不会改变执行结果的指令。删除它，或改成明确、可验证的约束。
