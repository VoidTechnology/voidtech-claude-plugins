# 设计 Brief 输出格式

brief 是给 claude.ai/design 的唯一上下文，按下列六节顺序写。示例文案用目标产品的界面语言书写（如 PRD 约定繁体中文，示例文案即用繁体中文）。

## 文件头：用法说明

标题 + 引用块，说明两件事：整段粘贴进 claude.ai/design 作为风格锚点；之后逐页下达指令，只需说页面名。给出一条指令示例。

## §0 项目定位

- 一句话说清产品是什么、给谁用；容易混淆时补一句「不是什么」。
- 端口与最小视口、界面语言、主题明暗、设计语言基调（一句概括气质与唯一强调色）。
- 影响界面结构的体系约束（如两层账号体系、RBAC）。

## §1 设计系统（Design Tokens）

### 1.1 颜色

用一个 CSS 代码块表达两层结构：

```css
:root {
  /* 原始色板：品牌强调色阶 + 中性色阶 + 语义原始色，每组一行注释说明使用纪律 */
  /* ---- Semantic mapping（页面规格只允许引用这一层） ---- */
  /* --bg-*（canvas/surface/sunken/hover/brand/…-subtle）、--text-*、--border-*、--status-* */
}
```

强调色注释必须写明使用边界（只用于主按钮/焦点/链接/激活态等），这是风格不跑偏的关键约束。推导新增的色值在注释中标注来源逻辑。

### 1.2 字体

- 一行说明字体族：UI 字体、目标语言的系统回退栈、等宽字体及其适用对象（ID/单号/金额）。
- 一张表列字号 token：`Token | size / line-height | weight | 用途`。只保留目标产品用得到的档位。

### 1.3 间距 / 圆角 / 阴影

- 间距基准与档位一行列出；圆角按用途标注（按钮/输入、卡片、大面板、pill）。
- 阴影分级给出具体值，并写明层次哲学（如「优先用边框 + 背景色阶拉层次，卡片默认无阴影」）。

## §2 核心组件规范

逐组件一条加粗词条，格式：**组件名**：关键尺寸 + 引用的语义 token + 状态变化（hover/激活/禁用/错误）。最后一条写**布局骨架**：侧边栏宽度、顶栏高度、内容区内边距与最大宽度。

## §3 页面清单与逐页规格

- 开头一句：每页默认套用 §2 布局骨架，下面只写主区内容。
- 每页一个小节：`### P<n>. 页面名（REQ-xx~yy）`，正文用短列表描述主区结构（从上到下）、表格列清单、表单分组、关键交互态。状态类字段写明状态→语义色的映射。
- 不属于任何单页的非功能约束（脱敏、权限禁用态等）在本节末尾用引用块统一说明其设计表达。

## §4 出图顺序建议

- 编号列表给出建议顺序：先生成「设计系统 / 组件预览」页确认风格，再做定骨架的第一批页面，校准后批量出剩余页。
- 结尾给一条可复用的逐页指令模板，例如：

> 沿用本设计系统与组件规范，生成【页面名】，内容按 brief 对应段落。<主题>主题，<设计语言>风格，<界面语言>文案。
