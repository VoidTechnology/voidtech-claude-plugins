# 核心技能中文可读性审计

## 审查范围

- 日期：2026-06-26
- 范围：`voidtech-core` 发布的 22 个技能，以及这些技能直接引用的 Markdown 参考文件
- 目标：让团队成员首次阅读时能直接理解触发条件、执行步骤、完成标准和关键术语

## 表达规则

1. 优先使用常见、直接的中文，不用生硬直译、夸张比喻或只在上游语境中成立的口语。
2. 命令、frontmatter 字段、文件名、代码标识符和代码块保持英文。
3. Git、PR、PRD、ADR、TDD、API 等团队通用缩写保持稳定。
4. seam、adapter、depth、locality 等技能内部架构术语在首次出现时给出中文解释，后续保持一致。
5. 模板中要求原样发布到外部系统的英文内容保持英文，周围说明使用中文。

## 自动约束

`scripts/check-portability.sh` 会扫描发布技能，阻止本轮已经淘汰的生硬译法重新出现。该检查只约束明确的问题短语，不禁止正常技术词汇，避免把文案审查变成机械替换。

英文自然化参考、代码块、命令名、标签值和外部系统要求的固定模板不属于汉化缺口。
